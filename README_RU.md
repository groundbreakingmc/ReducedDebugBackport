# ReducedDebugBackport

📖 [English version (README.md)](README.md)

Небольшой плагин для Spigot/Paper **1.16**, который прячет невидимых игроков от
дебаг-рендерера `visualize_entity_supporting_blocks` — он есть у новых клиентов
Minecraft (1.21.x), заходящих на старый сервер через **ViaVersion**.

> ⚠️ **Важно:** `reducedDebugInfo` работает по принципу «всё или ничего». У каждого
> затронутого клиента вместе с рендерером пропадут **координаты в F3**,
> **хитбоксы (F3 + B)** и **границы чанков (F3 + G)** — отключить только один рендерер
> нельзя. Ограничивай область действия через `min-protocol-version` и право
> `reduceddebugbackport.bypass`.

## Проблема

Начиная с 1.21.x ванильный клиент вынес свои внутренние дебаг-рендереры в UI
(`F3 + F6` → *Debug Options* → *Debug Renderers*). Один из них,
`visualize_entity_supporting_blocks`, можно выставить в режим **Always** — и он
рисует маркер на блоке под каждой сущностью, **включая игроков под невидимостью**.
Рендерер берёт данные, которые у клиента и так есть, поэтому сервер не может просто
«отключить кнопку».

Ванильный рычаг против этого — флаг **`reducedDebugInfo`**
(геймрул *«Limits contents of debug screen»*). Когда он активен, клиент отключает
эти дебаг-рендереры. Флаг приходит клиенту в login-пакете, а в рантайме
переключается пакетом **Entity Status / Entity Event** (статус `22` = включить,
`23` = выключить), нацеленным на собственную сущность игрока.

## Совместимость

Работает на серверах Spigot/Paper **1.16 и новее**.

Геймрул `reducedDebugInfo` существует **с 1.8**, поэтому на любой версии, которую
поддерживает плагин, можно вместо него просто выполнить:

```
/gamerule reducedDebugInfo true
```

для того же эффекта, но **на весь сервер**. Плагин нужен только когда требуется
**контроль по игроку** — например, урезать F3 лишь новым Via-клиентам, у которых
реально есть рендерер `visualize_entity_supporting_blocks`, не трогая F3
(координаты, хитбоксы) у нативных игроков. Такую гранулярность даёт право
`reduceddebugbackport.bypass`; обычный геймрул так не умеет.

## Требования

- Сервер Spigot/Paper **1.16**.
- **ViaVersion** (используется для определения реальной версии протокола клиента).
- **PacketEvents** *или* **ProtocolLib** (для отправки пакета).

## Установка

1. Положи `ReducedDebugBackport-1.0.jar` в `plugins/` рядом с ViaVersion и
   PacketEvents (или ProtocolLib).
2. Запусти сервер один раз, чтобы сгенерировался
   `plugins/ReducedDebugBackport/config.yml`.
3. При необходимости поправь `min-protocol-version` и перезапусти.

## Конфигурация

`config.yml`:

```yaml
# Версия протокола клиента, НАЧИНАЯ с которой слать reducedDebugInfo.
# Клиентов НИЖЕ этого значения не трогаем (у них остаются координаты/хитбоксы в F3).
min-protocol-version: 774   # 774 = 1.21.11

# Писать в консоль каждую отправку флага.
log-actions: false

# Подробный лог: для КАЖДОГО игрока печатает определённую версию протокола
# и решение (SKIP / SENT / BYPASS).
debug: false
```

Частые номера протоколов:

| Протокол | Версия | Протокол | Версия |
|---|---|---|---|
| 754 | 1.16.5 | 767 | 1.21 / 1.21.1 |
| 763 | 1.20 / 1.20.1 | 768 | 1.21.2 / 1.21.3 |
| 764 | 1.20.2 | 769 | 1.21.4 |
| 765 | 1.20.3 / 1.20.4 | 770 | 1.21.5 |
| 766 | 1.20.5 / 1.20.6 | 772 | 1.21.7 / 1.21.8 |
| 773 | 1.21.9 | 774 | 1.21.11 |

## Команда

`/rdb [off]` — только для операторов. Диагностический хелпер:

- Печатает определённую версию протокола, порог, состояние bypass и «нормальное
  решение» (SEND / SKIP).
- Шлёт пакет **в обход порога**: `/rdb` — статус `22` (включить), `/rdb off` —
  статус `23` (выключить). Удобно убедиться, что сам пакет работает, независимо от
  определения версии.

## Право

| Право | По умолчанию | Эффект |
|---|---|---|
| `reduceddebugbackport.bypass` | `false` | Игрок игнорируется — `reducedDebugInfo` не шлётся, F3 остаётся полным. |

Право проверяется в момент применения (вход / респавн / смена мира), поэтому
изменение права для онлайн-игрока вступит в силу после его следующего респавна или
перезахода.
